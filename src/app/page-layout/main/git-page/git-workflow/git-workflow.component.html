Git general workflow

When working with Git in a team, following a structured workflow ensures smooth collaboration and code consistency.

1.Clone the Repository (Only Once)

  • #git clone downloads the full history of the project, including all branches and commits.
  • If it's your first time working with the project , start by cloning the remote repository to your local machine using command:
    #git clone http adress or SSH key
  • Then change directory ~cd folder_name
    (You can also check your current branch using #git branch)

2.Fetch and Update Your Local Branches

  • Before starting any new task, always ensure your local branches are up-to-date with the remote repository.
  • #git fetch downloads changes but does not apply them automatically — it's safe to use anytime.
    (Independent of the current branch - it downloads all changes from the remote repository across all branches, but does not merge them with the local ones.)

3.Switch between branches

WARNING! If there are unsaved changes in the working directory,Git will not let you switch (you need to commit or discard them).

You can switch between branches using #git checkout

  • #git checkout main switch to the main branch
  • #git checkout feature/authswitch to the feature/auth branch

4.Update Your Local Repository.

#git pull used to update your local branch with the latest changes from a remote repository. It combines two actions in one:

  • fetch – downloads new commits and data from the remote branch.
  • merge – automatically merges the remote changes into your current branch.

5.Create a branch and name it based on its purpose — such as feature, hotfix, or bugfix.

Always work in a separate branch when developing new features or fixing bugs! You can use two ways of creating a new branch:

  • First run #git branch branch-name to create the branch, then #git checkout branch-name to switch to it.
  • Use #git checkout -b branch-name to both create and immediately switch to the new branch.

6.Stage changes

After you make changes in code - add them to Index.

  • #git add . stages all new and modified files in the current directory and subdirectories for the next commit.
  • #git add index.html will add to stage only index.html file.

7.Commit the changes

The git commit command is used to save your staged changes to the local Git repository. Here's what it does:

  • #git commit -m "commit description here" takes all changes that have been staged
  • Creates a new commit object with these changes
  • Records your name, email, the current date/time, and your commit message
  • Updates the current branch's reference to point to the new commit.
