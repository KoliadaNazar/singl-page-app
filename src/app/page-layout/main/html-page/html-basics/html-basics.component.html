HTML Meta Tags

HTML meta tags provide metadata about a web page and are placed inside the <head> section of the HTML document. They do not display content on the page but help browsers, search engines, and other services understand the page better.

  • Improve SEO (Search Engine Optimization)
  • Ensure correct rendering on all devices
  • Provide useful information for social sharing

<head>
 <meta charset="UTF-8">
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
 <meta name="description" content="Learn about HTML meta tags.">
 <meta name="keywords" content=" web developer,HTML, CSS, Js, Angular, portfolio">
 <meta name="author" content="Nazar Koliada">
</head>

The <meta charset="UTF-8"> defines the character encoding for a web page. Here's what it does:

  • Tells the browser that the page uses UTF-8 encoding
  • Prevents display issues with special characters, emojis, and multilingual text
  • Should always be first in <head>

Viewport (for mobile devices)

  • Makes page mobile-friendly
  • width=device-width matches screen width
  • initial-scale=1.0 sets default zoom level

Page Description

  • Used by search engines in snippets
  • Should be unique for each page
  • Recommended length: 150-160 characters

Keywords

  • Specifies page keywords
  • Don't overuse (10-15 words max)

Author

  • Identifies content author
  • May include contact information

Robots

  • Controls whether the page should be indexed or not by search engines.
  • <meta name="robots" content="index, follow">

Document Structure

  • <header> </header> - Introductory content, logos, navigation
  • <main> </main> - Primary content of the document
  • <footer> </footer> - copyright, contacts

Sections

  • <article> </article> - Self-contained composition (blog post, news story)
  • <section> </section> - Thematic grouping of content
  • <aside> </aside> - Sidebar or tangential content
  • <nav> </nav> - Navigation linkss

Text

  • <h1> to <h6> - Hierarchical headings
  • <p> </p>- Paragraph
  • <blockquote> </blockquote>- Extended quotation
  • <code> </code> - Code snippet
  • <ol> </ol> - Ordered list
  • <ul> </ul> - Unordered lis
  • <li> </li> - List item

Why Semantic HTML Matters

  • Accessibility: Screen readers use semantic tags to navigate content
  • SEO: Search engines better understand page structure
  • Maintainability: Clearer code organization
  • Future Compatibility: Standards-compliant markup lasts longer

Best Practices for Semantic HTML

  • Start with structure before styling
  • Choose the most specific tag available
  • Maintain proper hierarchy (h1 > h2 > h3)
  • Validate regularly using W3C validator
  • Test with screen readers to ensure accessibility

Semantic HTML forms the foundation of modern web development. By properly using these tags, you create websites that are more accessible, maintainable, and search-engine friendly.
