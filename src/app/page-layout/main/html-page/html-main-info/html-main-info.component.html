Button vs Link

<button></button> VS <a>Link</a>

Buttons <button> trigger actions (like form submissions or JavaScript functions), while links <a> navigate to new locations (other pages or resources). Use buttons for interactions that change data/state on the current page, and links for moving between pages or external references.

When to Use Buttons

  • Form submissions (login, search, contact forms)
  • Triggering JavaScript actions (modals, dropdowns, carousels)
  • Changing UI state (toggle switches, filters, sorting)
  • Interactive controls (media players, accordions)
  • Destructive actions (delete, remove, cancel)

When to Use Links

  • Navigation between pages (menus, breadcrumbs)
  • Linking to external resources (documents, websites)
  • Anchor links (jumping to page sections)
  • Downloadable files (PDFs, documents)
  • Email/phone links (mailto:, tel:)

Form Elements

HTML forms are used to collect user input and sendit to a server. Forms contain various elements that let users enter data.

An HTML form is defined using the <form> element. It usually includes various input fields and a submit button:

<form action="/submit" method="post" target="_blank">
 <label for="name"> Name:</label>
 <input type="text" id="name" name="name" required>
 <label for="email"> Email:</label>
 <input type="email" id="email" name="email" required>
 <button type="submit">Submit</button>
</form>

Why It Matters

  • Improves accessibility.

Main Form Attributes

  • action: attribute defines the action to be performed when the form is submitted.
  • method: Defines how the data is sent (GET or POST).

    Tip: Always use POST if the form data contains sensitive or personal information!

  • target attribute specifies where to display the response that is received after submitting the form (_blank, _self ,_parent, _top)

HTML Input Types

Here are the different input types you can use in HTML:

  • <input type="button"> - defines a button;
  • <input type="text"> - defines a single-line text input field;
  • <input type="password"> - defines a password field;
  • <input type="submit> - defines a button for submitting form data to a form-handler;
  • <input type="reset"> - defines a reset button that will reset all form values to their default values;
  • <input type="radio"> - defines a redio button. Radio buttons let a user select ONLY ONE of a limited number of choices;
  • <input type="checkbox"> - let a user select ZERO or MORE options of a limited number of choices
  • <input type="datetime-local"> - specifies a date and time input field, with no time zone.
  • <input type="date"> - is used for input fields that should contain a date;
  • <input type="email"> - is used for input fields that should contain an e-mail address;
  • <input type="tel"> - is used for input fields that should contain a telephone number;
