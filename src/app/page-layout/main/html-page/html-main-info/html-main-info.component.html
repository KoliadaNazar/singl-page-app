Button vs Link

<button></button> VS <a>Link</a>

Buttons <button> trigger actions (like form submissions or JavaScript functions), while links <a> navigate to new locations (other pages or resources). Use buttons for interactions that change data/state on the current page, and links for moving between pages or external references.

When to Use Buttons

  • Form submissions (login, search, contact forms)
  • Triggering JavaScript actions (modals, dropdowns, carousels)
  • Changing UI state (toggle switches, filters, sorting)
  • Interactive controls (media players, accordions)
  • Destructive actions (delete, remove, cancel)

When to Use Links

  • Navigation between pages (menus, breadcrumbs)
  • Linking to external resources (documents, websites)
  • Anchor links (jumping to page sections)
  • Downloadable files (PDFs, documents)
  • Email/phone links (mailto:, tel:)

Form Elements

HTML forms are used to collect user input and sendit to a server. Forms contain various elements that let users enter data.
